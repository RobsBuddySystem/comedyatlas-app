COMEDY ATLAS.
PUBLIC API — NO KEY REQUIRED

A free, read-only API for stand-up shows worldwide

Same idea as Songkick or Bandsintown, for stand-up comedy: every endpoint below reads straight from the exact same public listings data that pariscomedy.com's own pages show — nothing more, nothing extra. Anonymous, no signup, no API key. Built for programmers, aggregators, and anyone building on top of what's already public.

Anonymous
no account, no key, no auth header
60 / min
generous per-IP rate limit
CORS open
call it directly from a browser

Base URL

https://atlas-api.pariscomedy.com/v1/

All endpoints below are relative to this base. Every response is JSON (UTF-8) except the two .csv/.ics export routes, which are plain text.

Endpoints

GET/v1/events

Upcoming stand-up shows. Filterable, paged.

paramtypemeaning
citystringcity name, case-insensitive (e.g. Paris)
date_fromdateISO date, inclusive lower bound on starts_at
date_todateISO date, inclusive upper bound on starts_at
free_onlybooltrue to return only free shows
orgstringorganizer slug (see /v1/orgs/{slug})
languagestringfilter by language code (e.g. en, fr, it), or unconfirmed for shows whose language hasn't been confirmed yet. Default: no filter — every language publishes, since language is a label here, never a gate; the site's own default visitor experience applies language=en client-side, not this API
limitintdefault 50, max 200
offsetintdefault 0
curl "https://atlas-api.pariscomedy.com/v1/events?city=Paris&free_only=true&limit=20"
curl "https://atlas-api.pariscomedy.com/v1/events?city=Barcelona&language=it"
GET/v1/cities

Every city COMEDY ATLAS currently tracks.

curl "https://atlas-api.pariscomedy.com/v1/cities"
GET/v1/venues/{slug}

One venue's public detail (name, address, coordinates, capacity).

curl "https://atlas-api.pariscomedy.com/v1/venues/le-kibele"
GET/v1/orgs/{slug}

One organizer/club's public profile.

curl "https://atlas-api.pariscomedy.com/v1/orgs/comedy-crush"
GET/v1/shows/{slug}

One recurring show's public detail (format, frequency, typical duration).

curl "https://atlas-api.pariscomedy.com/v1/shows/friday-night-standup"

Org bulk export — for festival & club programmers

Running a Fringe venue or a club and want your own listings out, in one file, for your own site or calendar app? These two routes return every upcoming show under one organizer, in full.

GET/v1/orgs/{slug}/events.csv

Every upcoming event for that organizer, one row each — title, dates, venue, price/free signal, ticket link.

curl "https://atlas-api.pariscomedy.com/v1/orgs/comedy-crush/events.csv" -o comedy-crush.csv
GET/v1/orgs/{slug}/events.ics

Same events as one RFC5545 calendar file (multi-VEVENT) — subscribe in Google Calendar, Apple Calendar, Outlook, or any calendar app that accepts an .ics URL.

curl "https://atlas-api.pariscomedy.com/v1/orgs/comedy-crush/events.ics" -o comedy-crush.ics

Rate limits & caching

60 requests/minute per IP, across all /v1/ routes combined. Over the limit returns 429; wait a minute and retry. Every response carries an ETag and a short Cache-Control: public, max-age=... — send back If-None-Match on your next request and get a cheap 304 if nothing changed.

What's never in a response

This API reads from the exact same public views the site itself renders from, so it is structurally incapable of returning a comic's or organizer's private contact details — no email, no phone number, no booker-only contact field, ever. Only what's already visible on a public page, plus the COMEDY ATLAS page URL and the click-tracked go.html ticket redirect.

Attribution

Free to use, no key, no approval needed. If you build something public with this data, we just ask for a visible "Powered by COMEDY ATLAS" credit linking back to https://pariscomedy.com/comedy-atlas/. Questions, or want to be featured? contact@comedyatlas.app.

This API serves public listings data only. It is not a booking or ticketing API — every ticket link routes to the organizer's own real ticket page (via ticket_url). No SLA is offered today; the 60/min limit and response shapes above are the current contract and will be versioned (/v1/, /v2/, ...) rather than silently broken.
